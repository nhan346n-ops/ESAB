== Tide considerations

=== Definition of surface references:

* *Chart datum/Zero hydro (Zh)*
+
Chart datum  is the water level that depths displayed on a nautical chart are measured from. A chart datum is generally derived from some phase of the tide.[1] Common chart datums are lowest astronomical tide[1] and mean lower low water. In non-tidal areas, e.g. the Baltic Sea, mean sea level (MSL) is used.[2]

* *Lowest astronomical tide (LAT)*
+
Lowest Astronomical Tide is defined as the lowest tide level which can be predicted to occur under average meteorological conditions and under any combination of astronomical conditions.

* *Mean sea level (MSL)*
+
Often shortened to sea level, Mean sea level is an average level of the surface of one or more of Earth's bodies of water from which heights such as elevation may be measured


=== Tide models

Tide prediction modules is based on two harmonic models :

* *Shom harmonic tide prediction*
+
Shom module is a harmonic model using shom constant harmonic files (not provided). This module is for internal use by Ifremer only.
[IMPORTANT]
.Surface references of harmonic tides
Prediction are relative to LAT surface

* *FES2014*
+
FES2014 is the last version of the FES (Finite Element Solution) tide model developed in 2014-2016. It has been developed, implemented and validated by the LEGOS, NOVELTIS and CLS, within a CNES funded project.
+
It is a tide harmonic model over a gridded resolution of the distributed package of 1/16° resolution
+
.citation if you need to refer to the tide model :
[FES2014 citation]
FES2014 was produced by Noveltis, Legos and CLS and distributed by Aviso+, with support from Cnes (https://www.aviso.altimetry.fr/.
+
[IMPORTANT]
.Surface references of FES2014
====
Prediction of FES2014 api are relative to MSL.

LAT (rel MSL) and MSL (rel Ellipsoid) are used to compute prediction in LAT referential by default
====

=== Surface reference
Prediction module refers to different sea surfaces (Zh,LAT,MSL)
In order to be comparable and applied to dataprocessing, the process allow to change surface reference system.

This change in surface reference system is possible thanks to the use of the following reference datasets :

* *Mean Sea Level with respect to ellipsoid* :
+
The surface use to have a Mean Sea Level is retrieved from Aviso, it is a mean sea profile above a reference ellipsoid (T/P or WSG84).
+
This surface is available on a regular grid (1/60°x1/60°, 1 minute) computed using a 20-year of TOPEX/POSEIDON, ERS-2 , GFO, JASON-1,JASON-2, ENVISAT mean profile and data of the ERS-1, Jason-1, and Cryosat-2 geodetic phase
+
https://www.aviso.altimetry.fr/en/data/products/auxiliary-products/mss.html

[WARNING]
No garantee is given that this surface match the zero reference surface of FES2014

* *Lowest Astronomical Tide with respect to MSL*

This surface is computed by running FES2014 tide prediction over a period of 18 years (2003-2020) on a grid definition of 1/16. The minimum value of predictions is then taken as lowest astronomical tide.

* *Chart Datum with respect to ellipsoid GRS80*
+
Chart datum definition is defined by the bathyelli surface reference
This surface is avalaible on shom data portal (http://data.shom.fr) as two dataset for Atlantic and Mediteranean region

=== Datasets and restrictions

Pyat will consider that all referenced dataset are deployed in a single directory
This directory is made of several subdirectories :

==== FES 2014
* The fes2014 directory contains the AVISO tide dataset that can be retrieved after registration on AVISO web site.
+
This dataset is made up by a set of files a given after download of dataset
+
image::tide/fes2014_datasets.png[]

* The reference_surfaces directory is made of a LAT model computed from Tide over 18 years. This model is stored in a file named `LAT_MSL_World_model_2003_2020.nc`. This model is at a 1/16 deg resolution


==== Shom
* BATHYELLI_ZH_ell_V1_1 or BATHYELLI_ZH_ell_V2 directory contains bathyelli surface reference

* The harmonic constant files are `.har` files, their file path is passed as a parameter to the processing software
