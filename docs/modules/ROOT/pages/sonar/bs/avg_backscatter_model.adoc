= Backscatter statistical model response

== Overview
For each sounder mode estimate backscatter angular average values per incidence angle

== Algorithm parameters

The computation parameters are :

* _Sounder type_ defines the sounder model and the format of the source raw file. Keep it to AUTO to automatically detect the sounder type.

* _Use snippets_ to recompute mean backscatter value by detection. Mean backscatter values by detection are stored in Sonar/BeamGroup1/Bathymetry/detection_backscatter_r variable in XSF file. This variable contains precomputed data and will be used to store corrected bs values. The raw snippets bs values can be used with this option to relaunch process from uncorrected values.

* _Use embedded sound velocity profile (use_svp)_ should be always checked. Deactivate this parameter can be usefull in case of corrupted or wrong profiles. Ray bending is no more used to compute seafloor incidence angle.

* _Recompute insonified area from seafloor incidence angle (use_insonified_area)_ applies correction depending on the effective incidence angle on the seafloor. A DTM is mandatory to properly achieve this step.

* _Remove angular compensation (remove_compensation)_ specifies if the angular compensation (Lambert or other) should be removed from raw values. Use this option to work on uncorrected data (default true).

* _Remove calibration_ specifies if the embedded calibration should be removed from raw values (works for .kmall only). Use this option to work on uncalibrated data, in case of suspicious calibration, or to build a model in the goal of generating a new bscorr file from default calibration (with zeros).

* _Integration method_ defines the method used to compute mean backscatter values per incidence angle. Two methods are available: MEAN and MEDIAN. The MEAN method computes the average backscatter value for each incidence angle bin, while the MEDIAN method computes the median backscatter value for each bin, which can be more robust to outliers. Usually, the MEAN is used for calibration.

* _Linear scale for mean computation (linear_scale)_ specifies if the mean or median backscatter values are computed on AMPLITUDE (stem:[20log_{10}]) or ENERGY (stem:[10log_{10}]) then converted back to dB. Usually, the mean is computed on ENERGY for calibration.

* _Reference Dtm (i_dtm)_ specifies the .dtm.nc file used to estimate incidence angle with the seafloor by detection.

* _Input BSAR file (i_meanmodel)_ specifies an optional BSAR file containing absolute backscatter measurements by incidence angle, It could be acquired with calibrated sounder (EK80), or estimated by fitting a gsab model after averaging all incidence curves. This file is used for calibration purposes to force backscatter response by incidence to known values.

== Algorithm steps

The full processing pipeline is fully detailed in the article "Acoustic backscatter processing in GLOBE: an open-source software for echo sounder data analysis" (https://doi.org/10.3389/frsen.2025.1574129)

=== Step 1 : BL0 Raw data decoding and georeferencing

Per-beam seafloor backscatter values are either retrieved directly from the vendor raw file (Kongsberg systems) or computed from raw snippets samples as in Reson systems. All these data are converted by GLOBE and stored
in the pivot SONAR-netCDF4 format, used by GLOBE backscatter processing as its working format. When per-beam values are computed by GLOBE, snippets are converted to a ping-beam 2D data matrix through a simple mean in amplitude linear space:

[stem]
++++
BS_{ping,beam} = 20 \log_{10} ( \frac{1}{N} \sum_{i=1}^{N} 10^{BS_i/20} )
++++

and associated to the corresponding beam-sounding geographic position.

=== Step 2 : BL1 correction for gains applied in reception (Kongsberg example)

The first processing step is to remove vendor-specific compensations. Depending on MBES manufacturers, real-time recorded BS data might be compensated to cope with incidence
angle dependence. This compensation should be removed to retrieve uncompensated backscatter values.
In the case of Kongsberg sounders, two terms are removed from BS, corresponding to specular and Lambert law correction. Following equation is used to retrieve raw BS from Kongsberg recorded snippets.

[stem]
++++
\displaylines{
ampSI = BS − 20 \log_{10} ( \frac{r_n}{r} ) - \left\{ \begin{array}{cl}
\left( 1 - \sqrt{(r - r_n)/(r_{co} - r_n)} \right).\Delta BS & : \ r_n \leq r \leq r_co \\
0 & : \ r \geq r_co
\end{array} \right.
\\
With :
\\
\Delta BS: \overline{BS_O} - \overline{BS_N}
\\
\overline{BS_O}: \text{Mean backscattering coefficient for the oblique incidence region}
\\
\overline{BS_N}:\text{Mean backscattering coefficient near normal incidence}
\\r_n: \text{range at the normal incidence}
\\r_{co} = r_n / \cos{\theta_{co}}
\\\theta_{co}: \text{The crossover angle}
}
++++

In the case of Reson MBES (7150,7125), no real-time compensation is applied, so raw values from the snippets are used directly. However, without any details about how the insonified area is estimated, the next processing step is skipped, and consequently, we assume a worse correction.

The second processing step is to remove the vendor-specific
insonified area. Bottom target strength (BTS) is approximated from
the previously computed BS using the formula from
Lurton et al. (2015) :
[stem]
++++
BTS = BS + 10 \log_{10}(A)
++++

where A is the target insonified area as estimated in real time,
assuming that the bottom is perfectly flat for the whole swath and
perpendicular to the shortest range, called range to normal incidence stem:[r_n].

[stem]
++++
\displaylines{
A = \left\{ \begin{array}{cl} A_N : r \leq \sqrt{r^2_n + (\frac{cT_p}{2\Omega_{rx}})^2} \\
A_O : \text{otherwise}
\end{array} \right.
\\A_N=\Omega_{rx}\Omega_{tx}r^2
\\A_O=\frac{cT_p}{2\sqrt{1-r^2_n/r^2}}\Omega_{tx}r
\\\Omega_{tx}:\text{transmit beam opening}
\\\Omega_{rx}:\text{receive beam opening}
\\c:\text{sound celerity}
\\T_p:\text{pulse length}
}
++++

This estimation should be removed to retrieve an estimation
without vendor compensation. At the end of this stage, assuming no
correction on vendor computed transmission losses, the backscatter
level is the level received at the transducer face prior to any
application of gain follow and noted BL1 in the backscatter level
notation (Schimel et al., 2018).


=== Step 3 : Reestimation of the insonified area

Given beam pointing angles, sound velocity profiles, and slopes
from a processed bathymetry digital terrain model (DTM), full
seafloor incidence angles (stem:[\theta_i]) between the incident ray and the
seafloor normal are computed for each detection point. The
projections of these vectors onto the vertical plane, taken in the
across-track and along-track directions, yield the across-track
incidence angle stem:[\theta_{iy}] and the along-track incidence angle stem:[\theta_{ix}],
respectively. The orientation of the incidence plane normal
relative to the across axis gives us the seafloor aspect angle stem:[\theta_{ia}].
The insonified area is either limited by the pulse length (normal
incidence) or by beam width (oblique incidence), and the minimum
of these two scenarios is retained

[stem]
++++
\displaylines{
A_N = \frac{\Omega_{rx}\Omega_{tx} r^2}{\cos{\theta_{ix}}\cos{\theta_{iy}}}
\\A_O = \frac{c T_p}{2 \sin{\theta_i}} r \sqrt{(\Omega_{tx}\sin{\theta_{ia}})^2 + (\Omega_{rx} \sin{\theta_{ia}})^2}
\\A = \min(A_N, A_O)
}
++++

It is assumed that these formulas are simplified versions of the
real areas and are built by generalizing the previous ones to take into
account seafloor slopes in both directions and the tilt angle of the
sensor (Hammerstad, 2000; Lurton et al., 2015, Equation 25)
We have now :

[stem]
++++
BL2 = BL1 - 10\log_{10}A
++++

=== Step 4 : Angular response analysis

The next step is to determine and remove the angular response
of the transducers and the seafloor. To achieve this, the current
methodology is statistical and aims at separating the contributions of
the transducer (beam pattern and source level) from the
contribution of the seafloor type and geomorphology. The main
problem to be solved here is that the two phenomena are entangled:
the variation of the seabed backscattering strength, with the
incidence angle, disturbs the measurement of the beam patterns,
which vary with the transmission angles.
The static method relies on statistics computed on the largest
possible amount of data. Given this set of data, ping-beam detections
are sorted by incidence angle to apply a mean/median in linear units
on backscatter values with bins of 1° that lead to a statistical model of
BS response by incidence (stem:[meanBS_{incidence}]). The values of this model are then removed from the BS level by detection to get the residual
BS, defined as the difference between the actual BS value and the BS
model by incidence angle. Applying the same statistics by
transmission angle, we obtain a residual BS model by
transmission (stem:[meanBS_{residual}]). The transmission angle is
considered to be the beam-pointing angle expressed in the vessel
referential. The more variety and combinations we have in data
between angles of incidence and transmission, the more the models
will be decorrelated from each other, and the residual BS model will
be close to a beam pattern estimation.
As the exact characteristics of each hardware and acquisition
mode are not known or documented, the algorithm computes these
statistics by acquisition mode, by emission sector, and by reception
antenna. The analysis data are then written in a netCDF file with the
extension “bsar.nc” for backscatter angular response. The curves
contained in the bsar net.CDF file can be edited in GLOBE for
quality control and outlier filtering. Because data used in this step depend on
seafloor diversity, the algorithm considers a geographic mask to
restrict the model to a specific part of the data, carefully chosen
by the user.

=== Notes and recommendations
* Use MEDIAN for robustness when data contains spikes or strong outliers; use MEAN for calibration where energy conservation is important.
* Prefer ENERGY linear-scale averaging for physical correctness when merging powers.
* Ensure consistent handling of removed compensations so results are comparable between sessions.

=== Pulse length retrieval for insonified area computation

It is essential to retrieve the correct pulse length used during data acquisition to ensure accurate backscatter calculations.
In a general case, three values can be considered:

* Total pulse length (stem:[T_{p,total}]): duration between the start and the end of the pulse including any ramp up/down time.
* Effective pulse length (stem:[T_{p,eff}]): duration of the main part of the pulse where most energy is concentrated.
* Nominal pulse length stem:[T_{p,nominal}]: theoretical pulse length based on frequency bandwidth.

==== Kongsberg .kmall files
Pulse length is directly retrieved from the .kmall file as the effective pulse length, in the /Sonar/Beam_group1/Vendor_specific/effectiveSignalLength_sec variable, in seconds.

==== Kongsberg .all files
The pulse length used by Kongsberg in realtime is stored in the runtime parameters of the file.
However, for EM122, EM302 and EM710, the pulse length is not directly available in the .all file for the different sectors. In this case, the nominal pulse length is used: stem:[T_p = \frac{1}{BDW}] with stem:[BDW] the frequency bandwidth of the emitted pulse.
For EM2040, the used pulse length depends on the firmware version. For firmware versions prior to 2019, the used pulse length fits the nominal. While later, the effective pulse length is used. That's why, the runtime information is always preferred when suffisant (ie. all sectors share the same pulse for a given mode).

==== Reestimation of the insonified area
For reestimation of the insonified area, the effective pulse length is used with the following relations:
[stem]
++++
\displaylines{
\text{CW mode}:
\\T_{p,nominal} = 0.7*T_{p,total}
\\T_{p,eff} = 0.375*T_{p,total}
\\
\\ \text{FM mode}:
\\ T_{p,eff} = T_{p,nominal} = \frac{1}{BDW}
}
++++
