= Horizontal Slicer

== Overview

The horizontal slicer is a tool allowing to create g3d file containing horizontal slices of water column data. A slice is a mean or a max of values located around the location of the slice. each slice is represented as a lat/long regular grid of values.

The g3d file can be imported in globe and slices can be displayed and selected in 2D and 3D views.

== Algorithm parameters

The computation parameters are :

* _Vertical spacing_ (delta_elevation) specify the vertical distance between each slice. It is equal to the slice thickness. Compute button retrieves a vertical distance based on the max sample interval.

* _Grid count_ specify the expected number of slices. Spacing between them is computed with the min/max elevations for the file.

* _Vertical offset_ specify the vertical shift applied on slice positions. Set a positive value to move upward. Usefull to start the first slice with a gap above the seafloor.

* _Vertical reference_ specify the origin elevation surface used to generate slices.
** char_datum : slices are reference from the chart datum using the tide information included in xsf files. If no tide correction has been applied, reference is the sea surface.
** sea_floor : slices are reference from the seafloor. Seafloor is computed from the bathymetry group included in xsf files.
* _WC Filters_ can be set to discard specific echoes in the water column. It can be usefull to activate the bottom detection to remove all data around and below the seafloor.
