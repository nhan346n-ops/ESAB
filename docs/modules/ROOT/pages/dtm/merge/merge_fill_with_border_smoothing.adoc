= Merge Fill with border smoothing

== Implementation details

Merge fill is an algorithm taking a dtm as a reference and iteratively filling area with no data with others dtms. Operations are done one by one and when two datasets are merged they become the new reference dataset.

In order to smooth transition between two dtms when filling the no-data area, Globe allows to use a smoothing algorithm that works on the border between the datasets, this is called border smoothing. To give the best transition between the dataset, the algorithm will erase all data at the border of the reference and use some interpolation methods to create a transition area between the two datasets (i.e the fill gap algorithm for this first release, but other algorithms might be used later).

Note : When no interpolation inside the border is possible due to missing values surrounding the area, the algorithm will perform a classical merge fill (ie take the values without smoothing)


== Other layers behaviour

Other layers are computed regardless of the border.
•	This mean that CDI on the border is attributed to the secondary file
•	Interpolation flag is set along the border (since a fill gap is done)
•	Min an max values are the one extracted from file A
